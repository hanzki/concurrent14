In this assignment, you will implement an interprocess communication mechanism and use it for a simple chat system. In the following, "should" is used to denote a weak requirement; something characteristic of a good solution. "must" and "may not" denote strong requirements; failure to achieve these in your own code may cause your assignment to be failed; conversely, you may assume that all requirements on other code marked using these always hold.

Background

You have been assigned to write a prototype for a new platform-independent chat system that is intended to allow users to participate in a conversation from a wide range of devices. To make it easier to balance the load between servers and make the system less dependent on a specific communcations mechanism, your boss has decreed that you use tuple spaces to distribute messages between threads and the various servers in the system. Your job is to implement a basic tuple space system (the design team is still arguing about which tuple space implementation to use) and get the message distribution part of the chat system running.

Task descriptions

Your task is to write a simple tuple space implementation using standard Java synchronisation mechanisms and use it for the message passing part of a simple chat system. These are two separate tasks; your tuple space implementation must work with any compliant chat system implementation and vice versa. Both tasks must be completed well to pass the assignment.

The tasks are described on separate pages:

  • Task A: Tuple space
  • Task B: Chat system

General requirements

Specifications and concurrency

Your code must work exactly as described in the assignment specification and in the task descriptions. Any deviations may cause you to fail the assignment! If you're not sure about something, ask in the course newsgroup.

Perfectly correct concurrency in your code is required. Design your code before writing it, and try to keep the concurrency simple. Read your code through line-by-line and examine if there is any concurrent execution elsewhere that may cause problems. Even minor concurrency problems in your code may cause you to get a lower grade or fail the assignment.

Efficient use of concurrency is encouraged. Inefficient use of concurrency constructs will result in a lower grade.

Style requirements

The program should be written in clear, reasonably object-oriented Java. We will not accept programs that are hard to read, violate basic object-oriented programming principles or contain a large amount of repeated code with minor variations. Cryptic method and variable names should not be used.

Explanatory comments are required for code that is not self-explanatory to a programmer fluent in Java. However, you should not clutter your program by stating the obvious. Javadoc comments describing the required methods are unnecessary but tolerated. Also, please bear in mind that your programs will be read by people who are familiar with the specification; there is no need to repeat the requirements in your comments.

Language

Comments, variable and method names and reports may be written in English.

Plan

You are required to submit a plan for how you intend to implement the required functionality in all parts of the project. In particular, you should explain, at least in terms of pseudocode, the synchronisation and waiting algorithms you intend to use to implement the tuple space operations as well as how you use the tuple space to transfer messages in the chat system. The plan should not exceed 2-4 A4 pages of 10-12 point text. Please focus on explaining your reasoning rather than what your solution does in detail. Refer to the specification when appropritate, but do not needlessly repeat it (e.g. your plan should not be a verbatim "rewrite" of the specification).

Report

Your submission must contain a report describing briefly:

  • The reasoning behind your solution, in particular:
    • how the tuple space ensures correctness
    • how the chat system ensures that messages are not lost or duplicated
    • how the message order is kept consistent across listeners
  • A list of all the tuple types used an a description of what they are used for
  • What measures you have taken to ensure your solution is correct (e.g. model checking, proofs, testing).
  • Any unexpected behaviour you may have encountered during testing and how you have investigated and resolved it.

The report should not exceed 2-4 A4 pages of 10-12 point text, excluding diagrams and other graphics. Please focuson explaining your reasoning rather than what your solution does in detail.
